5W1H – How to Plan, Execute and Review Tasks

- What: What needs to be done? Example: "Stabilize dashboard routing and add dashboard tests".
- Why: Rationale for the task; alignment to ISO/PDCA.
- Who: Owner/role responsible (e.g., Backend Eng, QA, DevOps).
- Where: Scope and location in system (e.g., server.js, routes/dashboard-api.js, public/dashboard-*.html).
- When: Timeline, milestones, due dates.
- How: Steps to accomplish; references to scripts, commands, and templates.

Template usage
- For any task/PR, fill in 5W1H to provide clear context and enable fast review and repeatability.
