SOP: Incident Response

Goal
- Detect, respond, and recover from security incidents or critical system failures.

Roles
- Incident Lead, Security, DevOps, and Engineering.

Phases
- Preparation: runbooks and monitoring in place.
- Detection & Analysis: collect logs, alert triage, determine impact.
- Containment: isolate affected components, prevent spread.
- Eradication & Recovery: remove root cause, restore services, verify data integrity.
- Post-Incident: root cause analysis, lessons learned, process improvement.

Communication
- Stakeholders, customers if needed; incident channel defined in the runbooks.

Notes
- Keep evidence and logs for forensics; document changes and patch versions.
